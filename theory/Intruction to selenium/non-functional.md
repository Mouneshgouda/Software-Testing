# Non-Functional Testing

## What is Non-Functional Testing?

Non-Functional Testing is a type of software testing that evaluates how well a system performs rather than what it does.

It focuses on system attributes such as:

- Performance
- Security
- Reliability
- Scalability
- Usability
- Stability

### Simple Definition

> Non-Functional Testing checks the quality, speed, security, and reliability of a software application.

---

## Examples

- How many users can access the system simultaneously?
- How quickly does the application respond?
- Is user data secure?
- Can the system handle heavy traffic?

---

## Non-Functional Testing Process (Steps)

### Step 1: Identify Quality Requirements
- Determine performance, security, and usability expectations.

### Step 2: Define Test Metrics
- Response time
- Throughput
- CPU usage
- Memory usage

### Step 3: Design Test Scenarios
- Create load, stress, security, and usability scenarios.

### Step 4: Set Up Test Environment
- Configure tools and infrastructure.

### Step 5: Execute Tests
- Run performance, security, or load tests.

### Step 6: Analyze Results
- Measure system behavior and identify bottlenecks.

### Step 7: Optimize and Retest
- Fix issues and rerun tests.

---

## Example

### Scenario
E-Commerce Website Performance

### Test

- Simulate 5,000 concurrent users.

### Expected Result

- Pages load within 3 seconds.

### Actual Result

- Average response time = 2.5 seconds.

### Result

- Pass ✅

## Types of Non-Functional Testing

### Performance Testing
Checks application speed and responsiveness.

### Load Testing
Checks behavior under expected user load.

### Stress Testing
Checks behavior beyond normal capacity.

### Security Testing
Identifies vulnerabilities and security risks.

### Usability Testing
Evaluates user-friendliness.

### Compatibility Testing
Checks application across different devices, browsers, and operating systems.

### Reliability Testing
Ensures consistent performance over time.

### Scalability Testing
Determines how well the application handles growth.

---

## Benefits of Non-Functional Testing

- Improves application performance.
- Enhances security.
- Ensures system stability.
- Supports scalability.
- Improves user experience.
- Reduces production failures.

---

# Functional vs Non-Functional Testing

| Feature | Functional Testing | Non-Functional Testing |
|----------|------------------|------------------------|
| Focus | What the system does | How the system performs |
| Objective | Validate functionality | Validate quality attributes |
| Checks | Features and requirements | Performance, security, usability |
| Example | Login, Registration | Load Testing, Security Testing |
| Result | Feature works or not | System quality measurement |

---

# Quick Summary

## Functional Testing
**Question:** Does the feature work correctly?

Example:
- Login
- Signup
- Payment
- Search

## Non-Functional Testing
**Question:** How well does the feature perform?

Example:
- Performance
- Security
- Load
- Scalability
- Usability
